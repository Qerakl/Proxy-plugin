# Для разработчиков

## Архитектура

Два независимых расширения с общим UI и логикой:

| Папка | Назначение |
|-------|------------|
| `chrome/` | Сборка для Chrome (service worker, `proxy.settings`) |
| `firefox/` | Сборка для Firefox (event page, `proxy.onRequest`) |
| `lib/` | Общие модули (парсер, гео, тест, handlers) |
| `popup/` | HTML/CSS/JS интерфейса |
| `icons/` | Иконки 16/48/128 |

Браузер-специфичный код — только `chrome/background.js` и `firefox/background.js` + `manifest.json`.

## Синхронизация после правок

Если менял `lib/`, `popup/` или `icons/` в **корне** проекта:

```bash
./scripts/sync.sh
```

Скрипт копирует общие файлы в `chrome/` и `firefox/`.

Потом перезагрузи расширение в браузере.

## Локальная разработка

**Chrome:** загрузить папку `chrome/` в `chrome://extensions`

**Firefox:** загрузить `firefox/manifest.json` в `about:debugging`

## Проверка синтаксиса

```bash
node --check chrome/background.js
node --check firefox/background.js
node --check popup/popup.js
```

## native/

Скрипты relay (`native/proxy_relay.py`) — экспериментальные, в основном расширении не используются.

## Версии

Версию менять в обоих манифестах:

- `chrome/manifest.json`
- `firefox/manifest.json`

После правок в корневых `lib/` / `popup/` — запустить `sync.sh`.
